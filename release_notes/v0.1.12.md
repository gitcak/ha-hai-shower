# Hai Shower v0.1.12

## Fixes

- **Phantom "current session" after restart** — Restoring persisted usage
  records on integration startup (and processing each record during history
  sync) was writing the latest historical session's duration, volume, and
  session_id into the *live*-session sensors. The result: after every Home
  Assistant restart, `Current session duration` and `Current session water
  usage` displayed values from a prior shower that never cleared, even though
  the shower was idle. The live-session sensors are now driven only by live
  BLE telemetry and the real shower-end notify; restoration and history sync
  populate only the `Last session …` and cumulative sensors.

## Operational Notes

- No Home Assistant config migration is required.
- Restart Home Assistant after HACS updates the integration. After restart,
  `Current session duration` / `Current session water usage` will be
  unavailable until the next live shower starts (this is the correct
  behavior).

## Known Limitations

- Firmware version remains unavailable through ESPHome Bluetooth proxy
  security limits.
- App-side alert-setting changes are not yet read back into Home Assistant.
