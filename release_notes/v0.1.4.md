# Hai Shower v0.1.4

## Fixes

- Added the missing `unique_id_mismatch` abort string so reconfigure no longer shows a blank translation if you pick a different physical shower.
- Composite LED writes now default enable states to `True` to match the Hai app factory settings, preventing the shower LED from being silently disabled when only a threshold or color is changed. (Already fixed in the working tree, now promoted to release.)
- Downgraded the composite-write "assumed enable" log from `warning` to `debug` now that the assumption matches the device default.

## Security

- Diagnostics now redact the Bluetooth address, device ID, and device code via `async_redact_data` so users can safely paste diagnostics snippets into bug reports. `device_key` continues to be omitted entirely. Raw shower-end payload bytes are kept visible on purpose — they are protocol debug data and the primary reason anyone captures diagnostics.

## Operational Notes

- Restart Home Assistant after updating. No config migration required.

## Known Limitations

- Firmware version remains unavailable through ESPHome Bluetooth proxy security limits.
- App-side alert-setting changes are not yet read back into Home Assistant.
- Cloud history import is not implemented.
