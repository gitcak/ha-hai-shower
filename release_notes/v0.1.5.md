# Hai Shower v0.1.5

Re-cut of v0.1.4 plus one bug fix. v0.1.4 never shipped an artifact because
its release workflow failed on a missing announcement file.

## Fixes

- Statistics backfill: tolerate `sum=None` rows from `get_last_statistics`.
  Previously, if the recorder returned a row whose `sum` key existed but held
  `None`, the running-total accumulator would hit `TypeError` and silently
  abort the long-term-statistics import. Now coerced to `0.0`.
- All v0.1.4 fixes rolled forward: `unique_id_mismatch` abort string,
  composite LED writes defaulting enables to `True`, downgraded "assumed
  enable" log, diagnostics redaction via `async_redact_data`.

## Removed / Non-Goals

- `HaiCloudClient.get_shower_history` cloud-history stub — dropped.
- Firmware OTA / Nordic DFU — documented as a permanent non-goal. See
  `docs/roadmap.md` "Explicit Non-Goals".

## Operational Notes

- Restart Home Assistant after updating. No config migration required.

## Known Limitations

- Firmware version remains unavailable through ESPHome Bluetooth proxy
  security limits.
- App-side alert-setting changes are not yet read back into Home Assistant.
