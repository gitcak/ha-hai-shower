# Hai Shower v0.1.7

## Improvements

- Added a derived Status sensor with `idle`, `running`, `syncing`, and
  `unreachable` states so dashboards can show the shower lifecycle clearly
  while live BLE readings are unavailable.
- Added a diagnostic Last seen sensor that records the most recent successful
  BLE contact with the shower.
- Renamed live-only entities to make their scope clearer: Live water
  temperature, Live water flow rate, Current session duration, Current session
  water usage, and Battery voltage.
- Reduced normal idle BLE log noise. After a known shower has synced or been
  seen, expected sleep/offline refresh failures are logged at debug level
  instead of warning level.

## Operational Notes

- Restart Home Assistant after updating. No config migration required.
- Existing live telemetry sensors still become unavailable while the shower is
  asleep or unreachable; use the new Status and Last seen sensors for idle
  dashboard context.
- First-contact failures, history-sync failures, and user-initiated write
  failures still surface as warnings.

## Known Limitations

- Firmware version remains unavailable through ESPHome Bluetooth proxy security
  limits.
- App-side alert-setting changes are not yet read back into Home Assistant.
