# Hai Shower v0.1.11

## Fixes

- **Safer alert and LED writes during active showers** — Home Assistant now
  records threshold, LED color, and enable changes immediately, but defers BLE
  writes while live flow or runtime notifications are active. The integration
  flushes one composite `led_config` payload on the next idle refresh, avoiding
  the crash class seen when `led_config` was written repeatedly mid-shower.
- **RTC sync protocol alignment** — Ghidra/Hermes evidence showed
  `rtcSyncCharId` belongs to the 4-byte encrypted characteristic class. The
  integration now writes the UTC epoch as an XOR-encrypted UInt32LE payload
  instead of plaintext.

## Operational Notes

- No Home Assistant config migration is required.
- Restart Home Assistant after HACS updates the integration.
- Mid-shower alert setting changes may not appear on the physical LED until
  the next safe idle BLE refresh.

## Known Limitations

- Firmware version remains unavailable through ESPHome Bluetooth proxy security
  limits.
- App-side alert-setting changes are not yet read back into Home Assistant.
