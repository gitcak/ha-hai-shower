# Hai Shower v0.1.8

## Fixes

- Fixed HACS installs of v0.1.7 failing with "Integration 'hai_shower' not
  found." The release zip contents are now rooted at the integration files
  (`manifest.json` at the archive root), as required by HACS when
  `zip_release: true` is set. v0.1.7 zips wrapped everything under
  `custom_components/hai_shower/`, so HACS extracted the manifest one level
  too deep and Home Assistant could not locate the integration.

## Release packaging

- Each release now ships a single asset, `hai_shower.zip` (the filename HACS
  pins via `hacs.json`). The duplicate versioned `hai_shower-<ver>.zip` asset
  has been dropped.

## Operational Notes

- If you installed v0.1.7 via HACS, remove the integration in HACS, then
  reinstall after updating. Restart Home Assistant after the reinstall. No
  config migration is required.
- Manual installers: unzip `hai_shower.zip` directly into
  `config/custom_components/hai_shower/` (the archive now unpacks flat into
  the integration directory).

## Known Limitations

- Firmware version remains unavailable through ESPHome Bluetooth proxy security
  limits.
- App-side alert-setting changes are not yet read back into Home Assistant.
