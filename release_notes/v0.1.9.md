# Hai Shower v0.1.9

## Fixes

- Added a short post-disconnect reconnect backoff for ESPHome Bluetooth proxy
  paths. This gives the proxy time to finish GATT teardown before the next
  `establish_connection` attempt, reducing `ESP_GATTC_OPEN_EVT` / `status=133`
  reconnect races after an intentional disconnect.

## Release packaging

- Keeps the corrected HACS zip layout from v0.1.8: the release asset is the
  single stable `hai_shower.zip`, with `manifest.json` at the archive root.

## Operational Notes

- No Home Assistant config migration is required.
- Restart Home Assistant after HACS updates the integration.

## Known Limitations

- Firmware version remains unavailable through ESPHome Bluetooth proxy security
  limits.
- App-side alert-setting changes are not yet read back into Home Assistant.
