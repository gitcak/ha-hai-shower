# Hai Shower v0.1.10

## Fixes

- **LED status light blank after device crash** — The Hai shower stores its
  alert/LED configuration in volatile RAM. A firmware restart (e.g. caused by
  rapid alert-setting writes while BLE notifications were active) wiped that
  config, leaving the physical status LED blank even though Home Assistant still
  showed the saved alert state. The integration now re-pushes the full alert
  config (thresholds, enable bits, RGB colors) to the device on every fresh BLE
  connection, so the LED is restored correctly after any reconnect.

## Operational Notes

- No Home Assistant config migration is required.
- Restart Home Assistant after HACS updates the integration.

## Known Limitations

- Firmware version remains unavailable through ESPHome Bluetooth proxy security
  limits.
- App-side alert-setting changes are not yet read back into Home Assistant.
